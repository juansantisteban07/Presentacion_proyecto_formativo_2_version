Quiénes somos

Somos el equipo desarrollador del proyecto MIRADENT. Nuestro proyecto consiste en un sistema web para la gestión de citas odontológicas, pensado inicialmente para un consultorio familiar. Nuestro objetivo es digitalizar procesos que actualmente se realizan de manera manual, mejorando la organización, la atención al paciente y el control administrativo.

Problemática

El reto que identificamos

Gestión manual

Muchos consultorios odontológicos pequeños todavía administran las citas mediante agendas físicas, llamadas telefónicas o mensajes por WhatsApp.

Esto genera

  • Pérdida de información
  • Citas duplicadas
  • Dificultades para consultar historiales
  • Administración poco eficiente del tiempo del odontólogo
Después de analizar esta situación, vimos la oportunidad de desarrollar una solución web sencilla que automatice estos procesos sin que el consultorio tenga que invertir en software complejo o costoso.
Nuestra Solución

Una sola plataforma, tres roles

Nuestra propuesta consiste en desarrollar un sistema web que permita administrar pacientes, odontólogos, servicios y citas desde una sola plataforma.

Paciente
  • Registrarse
  • Agendar citas
  • Consultar sus citas
Administrador
  • Gestionar pacientes
  • Gestionar citas
  • Gestionar servicios
  • Gestionar odontólogos
Odontólogo
  • Consultar agenda
  • Ver citas
  • Registrar tratamientos
Modelado del Sistema

De la idea a la estructura de datos

El diseño de la base de datos se desarrolló siguiendo diferentes niveles de modelado para representar correctamente la estructura del sistema antes de su implementación.

Modelo Conceptual

Representa las entidades principales del negocio y la forma en que se relacionan entre sí. No incluye atributos, claves primarias, claves foráneas ni tipos de datos.

Modelo Lógico

Representa las entidades con sus atributos, claves primarias, claves foráneas y relaciones, definiendo la estructura lógica del sistema antes de la implementación física.

Modelo Entidad Relación (MER)

Representa la estructura relacional de la base de datos con sus entidades, relaciones y cardinalidades, sirviendo como base para la implementación del sistema.

Objetivo General

Desarrollar un sistema web que optimice la gestión de citas odontológicas en un consultorio familiar mediante la digitalización de procesos administrativos y clínicos.

Objetivos Específicos

Cómo lo vamos a lograr

01

Caracterizar las necesidades y problemas del consultorio mediante el levantamiento de información y entrevistas con los interesados.

02

Estructurar los requerimientos operativos y flujos de información necesarios para la correcta gestión de los datos del consultorio.

03

Definir el nuevo modelo de asignación, control y seguimiento para optimizar las agendas y citas del consultorio.

04

Establecer los mecanismos de control que sustituyan la gestión manual para asegurar la reducción de errores en los procesos administrativos y clínicos.

¿Por qué es importante?

Aunque existen sistemas comerciales, muchos consultorios pequeños no pueden acceder a ellos por sus costos o por su complejidad. Nuestro proyecto busca ofrecer una alternativa sencilla, funcional y adaptada a las necesidades reales de un negocio familiar.
